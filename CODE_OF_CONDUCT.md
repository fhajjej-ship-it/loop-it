# Code of Conduct

Loop it is a small public developer tool. Keep project discussions useful, specific, and respectful.

## Expected behavior

- Be clear about problems and tradeoffs.
- Critique code, docs, and decisions without personal attacks.
- Assume public issues and pull requests may be read by new users evaluating the project.
- Keep reports actionable: include steps, commands, expected behavior, and actual behavior when possible.

## Unacceptable behavior

- Harassment, threats, or discriminatory language.
- Publishing private credentials, personal data, or confidential project material.
- Repeated low-signal comments that derail issue or review threads.

## Reporting

Open a GitHub issue if the concern is about project behavior, docs, install, or safety. For sensitive security issues, follow `SECURITY.md`.
